前言

Hugo 是一個由Golang實現的靜態網站生成器(Static Site Generator, SSG)，不像WordPress, Ghost 和 Drupal 要花較多時間建立頁面，還有後續動態頁面更新的問題及安全性等隱憂。由於不是每個網站都需要 Server 來動態生成內容，串接資料庫等需求，像是企業網、筆記文件、Blog等，SSG 就是它實用的地方。

而 SSG 的優點在於：

快速部署，維護簡單，可拓展性高，高效能返回已渲染好的網頁

目前市面上較流行使用的 SSG 有 Jekyll , Hugo。其中 Jekyll 是以 RubyGems 環境，對 Windows 的使用者非常不友善QQ，在安裝前置作業很花時間。

而 Hugo 優點是它自動生成網頁的速度非常快，學習起來也相較容易，所以我直接選擇較好入門的 Hugo 和 Github Pages 搭建自己的 Blog。

其實是之前試著玩 Jekyll 安裝作業太複雜，覺得青春歲月就這樣去了……



Windows 安裝

Hugo 官網有提供教學影片可參考：Install Hugo | Hugo

建立資料夾

  1. 建立 C:\Hugo\bin 作為儲存執行文件，也可以放在其他地方
  2. 建立 C : \Hugo\Projects\blog 作為新專案目錄

下載 Hugo

  1. 下載最新 version 的壓縮文件 Releases · gohugoio/hugo · GitHub
  2. 解壓縮內容至剛剛建立的 C:\Hugo\bin
  3. 若 hugo 執行檔將被命名為 hugo_hugo-version_platform_arch.exe，請將文件重新命名為 hugo.exe 使用

將 Hugo 添加到的 Windows 環境變數設置中

  1. 開始［搜尋］檢視進階系統設定 > 進階 > 環境變數 > 使用者變數 > Path 編輯
  2. 新增路徑 C:\Hugo\bin 確定





開始建站

新增一個 site project

hugo new site mySite


Hugo 會自動在目錄中生成一個網頁結構：

|-archetypes  // 儲存.md的模板文件 (優先權高於/themes)
|-config.toml // 全局設置文件，默認下 FrontMatter 格式為.toml
|-content　　　// 儲存 about, post.md 文件
|-data		 // 數據文件，很少用
|-layouts	 // 模板文件
|-static	 // 靜態資源(img, css, js...)文件會複製到/public(優先權高於/themes/static)
|-themes	 // 存放網頁主題
