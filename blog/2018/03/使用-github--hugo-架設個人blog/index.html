前言

Hugo 是一個由Golang實現的靜態網站生成器(Static Site Generator, SSG)，不像WordPress, Ghost 和 Drupal 要花較多時間建立頁面，還有後續動態頁面更新的問題及安全性等隱憂。由於不是每個網站都需要 Server 來動態生成內容，串接資料庫等需求，像是企業網、筆記文件、Blog等，SSG 就是它實用的地方。

而 SSG 的優點在於：

快速部署，維護簡單，可拓展性高，高效能返回已渲染好的網頁

目前市面上較流行使用的 SSG 有 Jekyll , Hugo。其中 Jekyll 是以 RubyGems 環境，對 Windows 的使用者非常不友善QQ，在安裝前置作業很花時間。

而 Hugo 優點是它自動生成網頁的速度非常快，學習起來也相較容易，所以我直接選擇較好入門的 Hugo 和 Github Pages 搭建自己的 Blog。

其實是之前試著玩 Jekyll 安裝作業太複雜，覺得青春歲月就這樣去了……

安裝 Hugo

Windows 安裝

Hugo 官網有提供教學影片可參考：Install Hugo | Hugo

建立資料夾

  1. 建立 C:\Hugo\bin 作為儲存執行文件，也可以放在其他地方
  2. 建立 C : \Hugo\Projects\blog 作為新專案目錄

下載 Hugo

  1. 下載最新 version 的壓縮文件 Releases · gohugoio/hugo · GitHub
  2. 解壓縮內容至剛剛建立的 C:\Hugo\bin
  3. 若 hugo 執行檔將被命名為 hugo_hugo-version_platform_arch.exe，請將文件重新命名為 hugo.exe 使用

將 Hugo 添加到的 Windows 環境變數設置中

  1. 開始［搜尋］檢視進階系統設定 > 進階 > 環境變數 > 使用者變數 > Path 編輯
  2. 新增路徑 C:\Hugo\bin 確定

Mac 安裝

相較起 Windows，超級簡單…

$ brew install hugo

確認安裝

Hugo 安裝完成後，可以執行 $ hugo version 來確認是否安裝成功。

基本指令

$ hugo version               # 查看當前版本
$ hugo help                  # hugo 指令目錄查看
$ hugo new site yourSite     # 新增一個網站 project
$ hugo new post/your-post.md # 新增一篇文章
$ hugo server                # 啟用 localhost:1313

詳細指令可參考官方文件：Basic Usage｜Hugo

開始建站

新增一個 site project

$ hugo new site mySite

Hugo 會自動在目錄中生成一個網頁結構：

|-archetypes  # 儲存.md的模板文件 (優先權高於/themes)
|-config.toml # 全局設置文件，默認下 FrontMatter 格式為.toml
|-content　　　# 儲存 about, post.md 文件
|-data		 # 數據文件，很少用
|-layouts	 # 模板文件
|-static	 # 靜態資源(img, css, js...)文件會複製到/public(優先權高於/themes/static)
|-themes	 # 存放網頁主題

新增一篇文章

$ cd 專案目錄/mySite
$ hugo new about.md     
$ hugo new post/first-post.md

執行完成後即可在/content中看到about.md和/post/first-post.md。而 Hugo 編輯文章使用的檔案格式為 markdown，可以找喜好的編輯工具來撰寫.md。這邊推薦兩個不錯用的 online Markdown tool :

  • Marxico : 可搭配 Evernote 同步更新
  • Editor.md : 中文化介面

Markdown 結構預覽

試著打開/content/about.md中查看檔案，會發現指令自動建立的內容中包含了文章基本資訊，如：生成日期、標題、是否為草稿等，內文可以從 — 分隔線以下開始撰寫。

如果習慣用 VS code 編輯文章，則可以使用 預覽功能 查看完成的視圖。

啟用 hugo 伺服器執行

文章新增完成要在 local 端預覽，執行：

$ hugo server -D

hugo server 會預留 1313作連接埠，可於瀏覽器查看 http://localhost:1313

然後發現…一片空白，什麼都沒有！！

這很正常，因為網站還沒有添加主題模板，所以內容當然是空無一物。接著會繼續操作為網站添加主題。

ps. hugo server -D 指令中的 -D 為 --buildDrafts 參數簡寫，可在 localhost 端中預覽文章 draft: true 的內容

添加主題模板

預設生成的網站是沒有任何主題模板的

這點跟 Jekyll 不一樣，Hugo 自動生成的網站是不具備任何的預設模板，但為了快速建立網站的模板，這裡就直接下載主題來使用囉！ 主題模板可以在此網址下載： Hugo Theme

如果想要理解 Hugo 本身的 Layout 運作則可參考此篇： 從Jekyll遷移到Hugo，Hugo不完全指南

主題下載

首先挑一個喜歡的主題模板放到專案/themes目錄，這邊使用了ananke這個主題。

$ cd themes
$ git clone https://github.com/budparr/gohugo-theme-ananke.git

如果沒有使用 git 可以到主題 gitHub站 上直接下載 zip 解壓縮到 /theme 即可。

主題預覽

$ hugo server --theme=ananke

此時輸入 hugo server 帶入 theme 指令，可以在 localhost 上看到剛剛建立的文章預覽了！當然也可以下載更多的主題來預覽，只要在--theme=your-theme-name改成欲套用的主題名稱即可。

設定 config.toml

一開始由 hugo 生成的 config.toml 文件設定上較精簡，可以將下載下來的anake/exampleSite/config.toml或其他主題模板中的config.toml複製覆蓋到專案目錄中，再進行更細部的設定。

以下僅做常用的設定介紹，更詳盡的設定參數請參考官方文件：Configure Hugo｜Hugo

# 網頁的標題
title = "Your Site Title"   
    
# 伺服器主機
baseURL = "https://example.com" 

# 語言編碼 (en, zh-tw... 台灣)
languageCode = "en-us"     

# 設定使用的主題 (default: ./theme/THEMENAME)
theme = "ananke"                

# 預設內容語言
DefaultContentLanguage = "en"   

# 分頁數
paginate = 10 (default: 10)

# 分頁路徑名稱
paginatePath = "page" (default: "page")

# 定義文章訪問路径
[permalinks]                    
  post = "/:year/:month/:title/"

# 定義導航路徑及名稱
[[menu.main]]
  name = "About"
  identifier = "about"
  url = "/about/"
  weight = 10                 # 決定排序順位

[[menu.main]]
  name = "Blog"
  identifier = "blog"
  url = "/blog/"
  weight = 20

# 自定義參數
[params]
  version = "3.0.x"
  userparams = "bobo"

網站發布

Hugo 架站最後一步就是輸出成打包好的文件，只要在專案根目錄下執行：

$ hugo

就會在根目錄中生成/public資料夾，原本存在根目錄/static中的資料夾(ex.static/images)，會直接生成在/public/images。

GitHub Pages 託管

GitHub 上的 GitHub Pages 服務可以在個人的 GitHub 上做免費的 SSL 靜態頁面託管，所以必須的前置作業：

  1. 電腦安裝 Git 2.8 以上的版本
  2. 擁有一個 GitHub 帳戶

若需要更詳細的教學可以參考官網：Hosting on GitHub｜Hugo

GitHub Pages 頁面類型

要 Push 檔案到 GitHub 前，需先在自己的 GitHub 建立新專案。GitHub Pages 有兩種頁面類型：

  1. 用戶個人主頁：https://<USERNAME>.github.io/
  2. 專案名稱頁：https://<USERNAME>.hithob.io/<PROJECT NAME>/

可以選擇偏好的方式來建立新的 Repository

New repository

首先在網頁右上點選 + > New repository

使用用戶個人主頁當頁面網址，在 Repository name 輸入：USERNAME.github.io； 若使用專案名稱當頁面網址，Repository name 則輸入：PROJECT-NAME

repository新增完成後，需修改專案根目錄 config.toml 中的 baseUrl 設定成當前使用的 gitHub pages 網址，並重新執行$ hugo生成/public

當然也可以直接在/public 生成前執行以下命令來修改站點的 theme 和 baseUrl

$ hugo --theme=your-theme-name --baseUrl="https://username.github.io/"

ps. 使用此方法並不會修改根目錄中 config.toml 的 theme 和 baseUrl 設定

將網站 push 到 gitHub

baseUrl 設定重後，接著進入/public 資料夾

# 新增.git
$ git init

# remote push 的目的地 (網址可從專案中的 Clone ore download 中複製)
$ git remote add origin https://github.com/username/username.github.io.git

# 加入所有檔案到下一個 commit
$ git add .

# commit 加入的檔案
$ git commit -m "first commit"

# local master push origin master(set the remote as upstream)
$ git push -u origin master

若無發生錯誤狀況，到這邊已經執行完 push 至 gitHub 上了，可以在 https://username.github.io/ 看到網站了！

補充：gitHub Pages Setting

如果使用專案名稱當作頁面網址，需要多一個步驟去設定 gitHub Pages 的開關。打開專案 repository 點選 Settings

找到 GitHub Pages 設定區塊，將 Source 設定為 master branch 後點按 Save

設定好後，網站就發布完成了。可以在藍色區塊看到發布的 url

結語

原本想說要找一個平台放學習筆記，順便經營一下自己的 gitHub，在這邊修修那邊改改的過程中，終於生出這篇文章。第一次寫筆記，光是找相關資料和構思內容就花了大半的時間，當然過程中是獲得不少 exp 啦！順便練習 .md 怎麼寫。不過真的寫筆記文章超級費神費時的

有考慮之後慢慢更新 blog 在加入 DISQUS 留言系統，希望自己可以更有動力更新就是了。
